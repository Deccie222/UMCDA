# 分析：将 min 方向转换为 max 方向（取负值）的后果

## 转换操作

```r
# convert_directions_for_rmcda() 第156-157行
if (d[j] == "min") {
  A_conv[, j] <- -A_conv[, j]  # 将 min 方向的准则取负值
}
```

## 对 PROMETHEE 算法的影响

### 1. **相对排序保持不变** ✅

**关键点**：PROMETHEE 算法基于**成对比较**（pairwise comparison），取负值**不会改变相对排序**。

**原理**：
- PROMETHEE 计算的是替代方案之间的**相对偏好强度**
- 如果 A 在原始数据上优于 B，取负值后 A 仍然优于 B（只是数值符号相反）
- **排序顺序完全一致**

**示例**：
```r
# 原始数据（min 方向：成本，越小越好）
Cost: A1=100, A2=200, A3=150
排序: A1 > A3 > A2  (A1 最好)

# 转换后（取负值）
Cost: A1=-100, A2=-200, A3=-150
排序: A1 > A3 > A2  (A1 仍然最好，排序不变)
```

### 2. **Net Flow 值的符号和大小会改变** ⚠️

**影响**：
- Net Flow 的**绝对值**可能改变
- Net Flow 的**符号**可能改变（如果 min 方向准则占主导）
- 但**相对大小关系**保持不变（排序不变）

**示例**：
```r
# 假设只有成本准则（min 方向）
原始 net.flow: A1=0.5, A2=-0.3, A3=0.1
转换后 net.flow: A1=-0.5, A2=0.3, A3=-0.1

# 排序保持不变：A1 > A3 > A2
# 但数值符号相反
```

### 3. **混合方向（min + max）的影响** ⚠️

**场景**：同时有 min 和 max 方向的准则

**影响**：
- Min 方向的准则取负值
- Max 方向的准则保持不变
- Net Flow 是**加权组合**，符号取决于哪个方向占主导

**示例**：
```r
# 两个准则：Cost (min, weight=0.6), Quality (max, weight=0.4)
原始数据:
  Cost:    A1=100, A2=200, A3=150
  Quality: A1=8,   A2=9,   A3=7

转换后:
  Cost:    A1=-100, A2=-200, A3=-150  (取负值)
  Quality: A1=8,    A2=9,    A3=7     (保持不变)

# Net Flow 是加权组合，符号取决于权重和数值范围
```

## 潜在问题和风险

### 1. **数值范围变化** ⚠️

**问题**：
- 如果原始数据都是正数，取负值后变成负数
- 可能影响 RMCDA 内部的归一化或其他数值计算

**影响**：
- PROMETHEE 算法内部会进行归一化：`(row - colMins) / (colMaxs - colMins)`
- 取负值后，最大值和最小值会互换，但归一化后的相对关系保持不变

**示例**：
```r
# 原始数据（min 方向）
Cost: [100, 200, 150]
归一化: (100-100)/(200-100)=0, (200-100)/(200-100)=1, (150-100)/(200-100)=0.5

# 转换后（取负值）
Cost: [-100, -200, -150]
归一化: (-100-(-200))/(-100-(-200))=1, (-200-(-200))/(-100-(-200))=0, (-150-(-200))/(-100-(-200))=0.5

# 归一化后的相对关系保持不变（只是顺序相反，但排序逻辑一致）
```

### 2. **原始数据包含负数的情况** ⚠️

**问题**：
- 如果原始 min 方向的数据包含负数，取负值后变成正数
- 可能改变数据的分布特征

**示例**：
```r
# 原始数据（min 方向，包含负数）
Profit: [-100, -50, -200]  # 负数表示亏损，越小（绝对值越大）越差

# 转换后（取负值）
Profit: [100, 50, 200]  # 变成正数，但排序逻辑正确

# 问题：如果原始数据范围是 [-200, -50]，转换后是 [50, 200]
# 数据分布可能改变，但相对关系保持不变
```

### 3. **数值精度问题** ⚠️

**问题**：
- 取负值操作本身不会引入精度损失
- 但如果后续计算涉及大数值，可能影响精度

**影响**：
- 通常可以忽略，因为 PROMETHEE 会进行归一化

### 4. **对结果解释的影响** ⚠️

**问题**：
- Net Flow 的数值和符号可能改变
- 用户需要理解这是基于转换后的数据计算的

**影响**：
- **排序结果正确**：相对排序保持不变
- **数值解释**：Net Flow 的绝对值可能不同，但排序逻辑一致
- **用户理解**：需要知道这是基于转换后的数据，但排序是正确的
