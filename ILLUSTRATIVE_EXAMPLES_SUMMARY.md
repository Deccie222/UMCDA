# mcdaHub Illustrative Examples Summary

本文档总结了 mcdaHub 包基于三个 toy datasets 的功能演示。详细的代码示例和完整实现请参考包的 vignettes。

## 一、数据集概览

### 1.1 cars 数据集

**应用场景**：汽车选择决策问题

**数据特征**：
- **替代方案**：5 款汽车（Economic, Sport, Luxury, TouringA, TouringB）
- **评估准则**：3 个（Price, Consumption, Power）
- **数据格式**：性能矩阵 + 权重向量 + 方向向量
- **特殊属性**：包含配置文件（profiles）用于分类任务

**准则说明**：
- Price（价格）：min 方向，越低越好
- Consumption（油耗）：min 方向，越低越好  
- Power（功率）：max 方向，越高越好

**功能演示范围**：
- TOPSIS 和 PROMETHEE 算法的排序、选择和分类任务
- 权重敏感性分析
- 多算法比较分析

### 1.2 supplier 数据集

**应用场景**：供应商选择决策问题

**数据特征**：
- **替代方案**：3 个供应商（Supplier_A, Supplier_B, Supplier_C）
- **评估准则**：3 个（Quality, Price, Delivery）
- **数据格式**：成对比较矩阵（pairwise comparison matrices）
- **方法要求**：AHP 方法专用数据

**准则重要性**：
- Quality（质量）：最重要准则
- Price（价格）：次重要准则
- Delivery（交付）：第三重要准则

**供应商特征**（基于成对比较）：
- Supplier_A：高质量、高价格、良好交付
- Supplier_B：中等质量、中等价格、优秀交付
- Supplier_C：低质量、低价格、交付较慢

**功能演示范围**：
- AHP 排序任务
- AHP 选择任务

### 1.3 employee 数据集

**应用场景**：员工绩效评估问题

**数据特征**：
- **替代方案**：4 名员工（Employee_1 到 Employee_4）
- **评估准则**：3 个（Productivity, Teamwork, Innovation）
- **数据格式**：成对比较矩阵 + 长格式判断数据（judge_alt_profile）
- **方法要求**：AHPSort 方法专用数据

**准则重要性**：
- Productivity（生产力）：最重要准则
- Innovation（创新）：次重要准则
- Teamwork（团队合作）：第三重要准则

**绩效类别**：
- Poor：低于平均绩效
- Average：达到基本期望
- Good：超出期望

**功能演示范围**：
- AHPSort 分类任务

## 二、功能演示分类

### 2.1 基础算法功能演示

#### 演示 1：TOPSIS 排序功能（cars 数据集）

**演示目的**：展示 TOPSIS 算法在排序任务中的应用

**功能展示**：
- 创建排序任务对象（TaskRanking）
- 使用 DeciderTOPSIS 求解
- 获得排序结果（ResultRanking）

**结果说明**：
排序结果包含每款汽车的 TOPSIS 分数（相对接近度）和排名。分数越高表示越接近理想解，排名越靠前。

**应用价值**：
展示了如何基于多准则对替代方案进行完整排序，适用于需要全面评估所有选项的场景。

#### 演示 2：PROMETHEE 排序功能（cars 数据集）

**演示目的**：展示 PROMETHEE II 算法在排序任务中的应用

**功能展示**：
- 创建排序任务对象
- 使用 DeciderPROMETHEE 求解
- 获得基于净流量（net flow）的排序结果

**结果说明**：
PROMETHEE 方法通过计算每对替代方案的偏好关系，得到净流量分数。净流量越高，表示该替代方案在成对比较中表现越好。

**应用价值**：
展示了基于偏好关系的排序方法，适用于需要考虑替代方案之间相对关系的场景。

#### 演示 3：AHP 排序功能（supplier 数据集）

**演示目的**：展示 AHP 算法在排序任务中的应用

**功能展示**：
- 使用成对比较矩阵创建排序任务
- 使用 DeciderAHP 求解
- 获得基于层次分析的排序结果

**结果说明**：
AHP 方法通过成对比较矩阵计算准则权重和替代方案优先级，最终得到综合分数。结果包含一致性比率（CI/RI），用于评估判断的一致性。

**应用价值**：
展示了基于专家判断的层次分析方法，适用于需要结构化决策过程的场景。

#### 演示 4：选择任务功能（cars 和 supplier 数据集）

**演示目的**：展示不同算法在选择任务中的应用

**功能展示**：
- 创建选择任务对象（TaskChoice）
- 使用不同算法求解
- 获得选择结果（ResultChoice）

**结果说明**：
选择任务从多个替代方案中识别最佳方案。虽然算法逻辑与排序相同，但结果解释聚焦于单一最佳选择。

**应用价值**：
展示了如何从多个选项中选择单一最佳方案，适用于决策场景。

#### 演示 5：分类任务功能（cars 和 employee 数据集）

**演示目的**：展示不同算法在分类任务中的应用

**功能展示**：
- 创建分类任务对象（TaskSorting）
- 使用 TOPSIS/PROMETHEE/AHPSort 求解
- 获得分类结果（ResultSorting）

**结果说明**：
分类任务将替代方案分配到预定义的类别中。cars 数据集使用 FlowSort 方法（TOPSIS/PROMETHEE），employee 数据集使用 AHPSort 方法。

**应用价值**：
展示了如何将替代方案分类到不同等级或类别，适用于绩效评估、风险分级等场景。

### 2.2 高级分析功能演示

#### 演示 6：算法比较分析（cars 数据集）

**演示目的**：展示如何比较不同 MCDA 算法的结果一致性

**功能展示**：
- 使用同一任务（cars）应用多个算法（TOPSIS, PROMETHEE）
- 使用 AlgoRankCorrelation 计算算法间的 Spearman 秩相关系数
- 获得相关性矩阵

**结果说明**：
相关性矩阵显示不同算法排序结果的一致性。高相关性（接近 1）表示算法结果相似，低相关性表示算法结果存在差异。

**应用价值**：
- 验证不同算法的一致性
- 帮助选择最适合的算法
- 理解算法差异的原因

**可视化输出**：
相关性矩阵可以直观展示算法之间的关系。

#### 演示 7：权重敏感性分析（cars 数据集）

**演示目的**：展示权重变化对排序结果的影响

**功能展示**：
- 使用 WeightPerturbation 进行 OAT（One-At-A-Time）权重扰动
- 生成多个权重扰动场景
- 分析排序稳定性

**分析内容**：
1. **排序轨迹分析**：展示每个替代方案在不同权重场景下的排序变化
2. **排序稳定性分析**：计算每个替代方案排序的标准差，评估稳定性
3. **相关性分析**：分析不同扰动场景之间的相关性

**结果说明**：
- 排序轨迹图显示替代方案对权重变化的敏感程度
- 标准差图识别最稳定和最不稳定的替代方案
- 相关性热图揭示权重变化模式

**应用价值**：
- 评估决策结果的鲁棒性
- 识别关键权重
- 理解权重不确定性对结果的影响

**可视化输出**：
- 排序轨迹图（trajectory plot）
- 排序标准差图（SD plot）
- 相关性热图（correlation heatmap）
